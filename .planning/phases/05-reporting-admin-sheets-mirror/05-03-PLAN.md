       - `sync_changed_emails(self)` -- main entry point:
         - Read `SystemConfig.get("sheets_last_synced", "")` as ISO datetime (or epoch if empty)
         - Query `Email.objects.filter(processing_status="completed", is_spam=False, updated_at__gt=last_synced)` -- all emails changed since last sync
         - If no changed emails, return early
         - Call `_ensure_tab_exists()` if not `_initialized`
         - Build row index cache if empty: read column A (message_id is NOT in columns -- use a hidden column or match by scanning). ACTUALLY: simpler approach -- add a 10th hidden column "Message ID" for lookups. Or: scan column C (Subject) + column B (From) for matching. Best approach: add Message ID as column 10 (hidden from casual viewers but needed for sync).
         - Revised: `COLUMNS = ["Date", "From", "Subject", "Inbox", "Category", "Priority", "Assignee", "Status", "SLA Deadline", "Message ID"]` -- 10 columns. Message ID in last column for programmatic lookup, humans ignore it.
         - Separate emails into "new" (message_id not in `_row_index`) and "existing" (message_id in `_row_index`)
         - Append new rows via `_append_rows()`
         - Update existing rows via `_batch_update_rows()`
         - Write `sheets_last_synced` to SystemConfig with current ISO timestamp

       - `_ensure_tab_exists(self)`:

         - Read column J (Message ID) from "v2 Mirror" tab
         - Build `{message_id: row_number}` dict
         - Called once on first sync, and on cache miss

       - `_email_to_row(self, email) -> list`:
         - Format: [received_at formatted "YYYY-MM-DD HH:MM", sender, subject, inbox, category, priority, assignee full name or "", status, sla_respond_deadline formatted or "", message_id]

       - `_append_rows(self, rows)`: Single `values().append()` call
       - `_batch_update_rows(self, updates: dict[int, list])`: Single `values().batchUpdate()` call

       - ALL methods wrapped in try/except at the `sync_changed_emails` level -- log warning, never raise

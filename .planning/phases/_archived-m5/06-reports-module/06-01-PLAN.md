Create `apps/emails/services/reports.py` with four aggregation functions. All accept `start_date`, `end_date`, and optional filter kwargs (`inbox`, `category`, `assigned_to`).

1. `get_overview_kpis(start, end, **filters)` -> dict with:
   - `total_emails`: Count of non-spam Emails received in period (filter by `Email.received_at`)
   - `avg_response_minutes`: Average minutes from Thread.created_at to first ActivityLog with action=ACKNOWLEDGED for threads in period. Use `Avg` with `ExpressionWrapper` and `F()` on ActivityLog. Return None if no data.
   - `sla_compliance_pct`: Percentage of threads where `sla_ack_deadline` is NOT breached (acknowledged before deadline OR still within deadline). Count threads with `sla_ack_deadline__isnull=False` in period. If zero threads, return 100.0.
   - `open_threads`: Count of Threads with status != "closed" (real-time, ignores date range).

2. `get_volume_data(start, end, **filters)` -> dict with `labels` (date strings) and `datasets` (list of dicts per inbox with `label`, `data`, `backgroundColor`):
   - Group non-spam Emails by `TruncDate('received_at')` and `to_inbox`
   - Daily granularity, one dataset per unique inbox
   - Fill in zero for dates with no emails (generate full date range from start to end)
   - Colors: use `#a83362` for info@ inbox, `#3b82f6` for sales@ inbox, `#10b981` for any other

3. `get_team_data(start, end, **filters)` -> dict with `labels` (user display names) and two datasets:
   - `handle_count`: Count of ActivityLog entries with action IN (ACKNOWLEDGED, CLOSED) per `performed_by` in period
   - `avg_response_minutes`: Average time from thread.created_at to first ACKNOWLEDGED ActivityLog per user
   - Only include active users with at least 1 action in period

4. `get_sla_data(start, end, **filters)` -> dict with:
   - `compliance_pct`: Same as KPI but standalone
   - `donut_data`: dict with `met` and `breached` counts for donut chart
   - `trend`: list of dicts `{date, compliance_pct}` -- daily compliance % over the period
   - `breaches`: list of dicts `{thread_id, subject, priority, deadline, actual, exceeded_by}` -- last 20 SLA breaches (threads where acknowledged_at > sla_ack_deadline)

All queries use Django ORM only (no raw SQL). Apply `is_deleted=False` filter on Thread/Email (SoftDeleteModel). Apply optional filters: `inbox` filters Email.to_inbox, `category` filters Thread.category, `assigned_to` filters Thread.assigned_to_id.

Add helper `_apply_filters(qs, model, **filters)` to DRY filter application across functions.

// Store chart instances
const charts = {};

// Destroy before creating new
function createChart(id, config) {
    if (charts[id]) { charts[id].destroy(); }
    charts[id] = new Chart(document.getElementById(id), config);
}

// Destroy all on HTMX navigation away
document.body.addEventListener('htmx:beforeSwap', function(e) {
    Object.values(charts).forEach(c => c.destroy());
});

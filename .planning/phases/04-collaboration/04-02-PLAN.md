  <behavior>
    - ThreadViewer with thread FK, user FK, last_seen DateTimeField creates and saves
    - unique_together on (thread, user) prevents duplicate viewer records
    - Viewers with last_seen older than 30 seconds are considered inactive
    - heartbeat endpoint updates last_seen for current user on a thread
    - heartbeat endpoint returns list of active viewers (excluding current user)
    - Opening a different thread clears old viewer record (or it expires via 30s timeout)
    - get_active_viewers(thread, exclude_user) returns users with last_seen within 30s
  </behavior>
  <action>
1. **ThreadViewer model** in models.py (after InternalNote):
   ```python
   class ThreadViewer(models.Model):
       thread = models.ForeignKey(Thread, on_delete=models.CASCADE, related_name="viewers")
       user = models.ForeignKey(settings.AUTH_USER_MODEL, on_delete=models.CASCADE, related_name="viewing_threads")
       last_seen = models.DateTimeField(auto_now=True)

       class Meta:
           unique_together = [("thread", "user")]

       def __str__(self):
           return f"{self.user} viewing {self.thread_id}"
   ```
   Note: NOT using SoftDeleteModel (viewer records are ephemeral, cleanup is fine).

2. **get_active_viewers(thread_pk, exclude_user_id=None)** utility function in views.py (or a small helper):
   - Cutoff: `timezone.now() - timedelta(seconds=30)`
   - Query: `ThreadViewer.objects.filter(thread_id=thread_pk, last_seen__gte=cutoff).select_related("user")`
   - Exclude the requesting user if exclude_user_id provided
   - Return queryset of ThreadViewer objects

3. **viewer_heartbeat(request, pk)** — POST endpoint in views.py:
   - `ThreadViewer.objects.update_or_create(thread_id=pk, user=request.user, defaults={"last_seen": timezone.now()})`
   - Cleanup stale records: `ThreadViewer.objects.filter(thread_id=pk, last_seen__lt=cutoff).delete()` (opportunistic cleanup)
   - Get active viewers (excluding current user)
   - Return rendered `_viewer_badge.html` partial (just the badge HTML, for HTMX swap)

4. **clear_viewer(request, pk)** — DELETE endpoint (called when navigating away):
   - `ThreadViewer.objects.filter(thread_id=pk, user=request.user).delete()`
   - Return empty response (204)

5. **URL patterns**:
   ```python
   path("threads/<int:pk>/heartbeat/", views.viewer_heartbeat, name="viewer_heartbeat"),
   path("threads/<int:pk>/clear-viewer/", views.clear_viewer, name="clear_viewer"),
   ```


# Phase 3 — UI Design Contract

> Visual and interaction contract for the mark-irrelevant feature. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (Django templates + HTMX 2.0) |
| Icon library | Inline SVG (heroicons stroke style, consistent with existing codebase) |
| Font | Plus Jakarta Sans (weights 300-800 loaded via Google Fonts) |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding (`py-0.5`) |
| sm | 8px | Compact element spacing (`gap-2`, `py-2`) |
| md | 16px | Default element spacing (`gap-4`, `px-4`) |
| lg | 20px | Section padding (`px-5` -- matches existing detail panel) |
| xl | 24px | Layout gaps (`gap-6`) |
| 2xl | 32px | Modal internal padding |
| 3xl | 48px | Not used in this phase |

Exceptions: Detail panel uses `px-5` (20px) consistently -- this phase follows that convention rather than strict 16px.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `bg-white` | Detail panel background, modal background |
| Secondary (30%) | `slate-50` / `slate-100` | Input backgrounds, muted containers |
| Accent (10%) | `primary-600` (#a83362) | Primary CTA buttons (not used in this phase -- mark-irrelevant uses amber) |
| Destructive | `red-600` | Spam actions only (not used for irrelevant) |

## Component Inventory

### 1. Mark Irrelevant Button (Detail Panel)

**Location:** Standalone section below AI triage area (after Draft Reply collapsible, before scrollable timeline).

**Structure:**
```
<div> section container (border-t, px-5 py-2.5)
  <button> "Mark Irrelevant" (amber, full icon+text)
</div>
```

**Specifications:**
- Button class: `inline-flex items-center gap-1 px-3 py-1.5 text-[10px] font-bold text-white bg-amber-500 rounded-md hover:bg-amber-600 transition-colors cursor-pointer`
- Icon: Slash circle (18.364 18.364 pattern), `w-3 h-3`, inline before text
- `hx-disabled-elt="this"` for loading state
- Click opens the reason modal (JS, not HTMX)
- Visibility: `{% if can_mark_irrelevant %}` -- gatekeeper/admin only, completely hidden from members

### 2. Reason Modal

**Trigger:** Button click in detail panel, or context menu selection (which opens detail + auto-triggers modal).

**Overlay:**
- `fixed inset-0 bg-black/50 z-50` with `backdrop-blur-sm`
- Click outside or Escape key closes modal
- Keyboard shortcut `I` opens modal (when not in input/textarea/select)

**Modal container:**
- `bg-white rounded-xl shadow-2xl border border-slate-200 w-full max-w-md mx-auto`
- Positioned: `fixed top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2` (centered)
- Padding: `p-6` (24px)
- Entry animation: `opacity-0 scale-95` to `opacity-100 scale-100` (150ms ease-out)

**Modal content:**
```
<h3> "Mark as Irrelevant" -- text-[14px] font-bold text-slate-900
<p> helper text -- text-[12px] text-slate-500 mt-1
<textarea> reason input -- mt-4, rows="3"
<div> button row -- mt-4, flex justify-end gap-2
  <button> "Cancel" -- slate secondary
  <button> "Mark Irrelevant" -- amber primary, disabled until textarea non-empty
```

**Textarea:** `w-full text-[12px] border border-slate-200 rounded-md px-3 py-2 bg-slate-50 text-slate-700 resize-none focus:ring-2 focus:ring-amber-400/30 focus:border-amber-300` (matches note textarea focus ring pattern but amber instead of amber -- consistent)

**Cancel button:** `px-3 py-1.5 text-[10px] font-bold text-slate-500 bg-slate-100 rounded-md hover:bg-slate-200 transition-colors cursor-pointer`

**Confirm button:** `px-3 py-1.5 text-[10px] font-bold text-white bg-amber-500 rounded-md hover:bg-amber-600 transition-colors cursor-pointer disabled:opacity-50 disabled:cursor-not-allowed`

**Form submission:** `hx-post` to mark_irrelevant URL, `hx-target="#thread-detail-panel"`, swaps entire detail panel on success.

### 3. Irrelevant Badge (Thread Card)

**Location:** Line 3 of thread card, alongside spam badge in the shrink-0 badge cluster.

**Specifications:**
- `inline-flex items-center px-1.5 py-0.5 rounded text-[8px] font-bold uppercase tracking-wider bg-amber-100 text-amber-700`
- Text: "Irrelevant"
- Title attribute: "Marked as irrelevant by [user]"
- Only visible when thread has IRRELEVANT status and is shown via status filter

### 4. Irrelevant Stat Card (Gatekeeper/Admin Only)

**Location:** After the "New" stat card in the stat card row, only visible to gatekeeper/admin users.

**Specifications:**
- Same structure as existing stat cards: `flex items-center gap-1.5 px-2.5 py-1.5 rounded-lg border ... hover:shadow-md transition-all cursor-pointer shrink-0`
- Colors: `border-amber-200/50 bg-amber-50/60 hover:border-amber-300`
- Icon: Slash circle, `w-3 h-3 text-amber-500`
- Count: `text-[12px] font-extrabold text-amber-700 tabular-nums`
- Label: `text-[9px] font-bold text-amber-500/70 uppercase` -- "Irrelevant"
- Links to: `?view=all_open&status=irrelevant`
- Active state: `ring-2 ring-primary-500/30` when `current_status == 'irrelevant'`

### 5. Context Menu Entry

**Location:** Status group (Group 3), after "Close" and before the spam divider.

**Specifications:**
- Same class pattern as all context menu items: `ctx-item flex items-center justify-between w-full px-3 py-1.5 text-[12px] hover:bg-slate-700/60 transition-colors`
- Icon: Slash circle, `w-3.5 h-3.5 opacity-60`
- Text: "Mark Irrelevant"
- Keyboard hint: `<kbd class="text-slate-500 text-[10px] font-mono ml-4">I</kbd>`
- Visibility: `{% if can_mark_irrelevant %}` -- completely hidden from members (same pattern as "Assign to...")
- Behavior: `onclick` opens detail panel via `hx-get` to thread detail, then JS auto-opens reason modal after HTMX swap completes


### 8. Revert to New Button (Irrelevant Thread Detail)

**Location:** Replaces the "Mark Irrelevant" button section when thread status is IRRELEVANT.

**Specifications:**
- Button class: `inline-flex items-center gap-1 px-3 py-1.5 text-[10px] font-bold text-slate-600 bg-slate-100 rounded-md hover:bg-slate-200 transition-colors cursor-pointer`
- Icon: Arrow-path/undo icon, `w-3 h-3`
- Text: "Revert to New"
- `hx-post` to revert_irrelevant URL, `hx-target="#thread-detail-panel"`
- `hx-confirm="Revert this thread to New? It will return to the triage queue."` -- native browser confirm (lightweight, no custom modal needed)
- `hx-disabled-elt="this"` for loading state

### 9. Status Filter Dropdown Option

**Location:** Existing status filter `<select>` in sidebar filter section.

**Specifications:**
- New `<option value="irrelevant">Irrelevant</option>` added after existing status options

## Interaction Contracts

### Modal Behavior

| Trigger | Action |
|---------|--------|
| Click "Mark Irrelevant" button | Open modal with backdrop, focus textarea |
| Press `I` key (not in input) | Open modal with backdrop, focus textarea |
| Context menu "Mark Irrelevant" | Load thread detail via HTMX, then auto-open modal after `htmx:afterSwap` |
| Press `Escape` | Close modal, return focus to trigger button |
| Click backdrop | Close modal |
| Submit with empty textarea | Blocked -- confirm button disabled until `textarea.value.trim().length > 0` |
| Submit with reason | HTMX POST, swap detail panel, show toast |

### State Transitions

| Current State | Action | Result |
|---------------|--------|--------|
| Any status (NEW/ACK/CLOSED) | Mark Irrelevant | Status = IRRELEVANT, hidden from default views |
| IRRELEVANT | Revert to New | Status = NEW, assigned_to = null, visible in triage queue |

### Toast Messages

Use the existing toast pattern (fade-out animation after 3s):
```html
<div class="px-5 py-2 bg-amber-50 border-b border-amber-200/60 text-xs font-medium text-amber-700"
     style="animation: fadeOut 1s ease-out 3s forwards;">
    Thread marked as irrelevant
</div>
```

    **Step 1: Extend _build_thread_detail_context in views.py**

    Inside `_build_thread_detail_context`, after computing `can_claim`, add three new context variables:

    ```python
    # Reassign candidates: users with CategoryVisibility for this thread's category
    # (only needed when member views their own thread)
    reassign_candidates = []
    is_own_thread = thread.assigned_to == request.user
    claim_disabled = False

    if is_own_thread and not request.user.can_assign:
        # Member viewing own thread -- compute reassign targets
        reassign_candidates = User.objects.filter(
            is_active=True,
            pk__in=CategoryVisibility.objects.filter(
                category=thread.category
            ).values_list("user_id", flat=True)
        ).exclude(pk=request.user.pk).order_by("first_name", "username")

    if thread.assigned_to is None and not request.user.can_assign:
        # Member on unassigned thread -- check if they have visibility
        has_vis = CategoryVisibility.objects.filter(
            user=request.user, category=thread.category
        ).exists()
        if not has_vis:
            claim_disabled = True
    ```

    Add `reassign_candidates`, `is_own_thread`, and `claim_disabled` to the returned context dict.

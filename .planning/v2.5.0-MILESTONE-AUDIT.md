## Integration Issues

### P0 — Pipeline ignores override flags (INTEL-11)

**File:** `apps/emails/services/assignment.py` lines 541–551
**Issue:** `update_thread_preview()` unconditionally overwrites `thread.category` and `thread.priority` from the latest AI triage, ignoring `category_overridden` and `priority_overridden` flags.
**Impact:** User-corrected category/priority silently lost when next email arrives in thread.
**Fix:** Check flags before overwriting:
```python
if not thread.category_overridden:
    thread.category = latest_triaged.category
if not thread.priority_overridden:
    thread.priority = latest_triaged.priority
```

### P1 — Auto-assign config key naming (INTEL-04)

**File:** `apps/emails/services/pipeline.py` lines 85–88
**Issue:** `auto_assign_confidence_threshold` implies numeric value but requires exact string `"HIGH"`. Default `"100"` intentionally disables, but an operator setting `"80"` would silently fail.
**Fix:** Rename to `auto_assign_confidence_tier` or add admin help text.

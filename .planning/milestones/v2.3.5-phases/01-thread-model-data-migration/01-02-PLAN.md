class Thread(SoftDeleteModel, TimestampedModel):
    class Status(models.TextChoices):
        NEW = "new", "New"
        ACKNOWLEDGED = "acknowledged", "Acknowledged"
        CLOSED = "closed", "Closed"

    gmail_thread_id = models.CharField(max_length=255, unique=True)
    subject = models.CharField(max_length=500, blank=True, default="")
    last_message_at = models.DateTimeField(null=True, blank=True)
    last_sender = models.CharField(max_length=500, blank=True, default="")
    last_sender_address = models.EmailField(blank=True, default="")
    status = models.CharField(max_length=20, choices=Status.choices, default=Status.NEW)
    assigned_to = models.ForeignKey(AUTH_USER_MODEL, null=True, blank=True, related_name="assigned_threads")
    assigned_by = models.ForeignKey(AUTH_USER_MODEL, null=True, blank=True, related_name="assigned_by_threads")
    assigned_at = models.DateTimeField(null=True, blank=True)
    sla_ack_deadline = models.DateTimeField(null=True, blank=True)
    sla_respond_deadline = models.DateTimeField(null=True, blank=True)
    category = models.CharField(max_length=100, blank=True, default="")
    priority = models.CharField(max_length=50, blank=True, default="")
    ai_summary = models.TextField(blank=True, default="")
    ai_draft_reply = models.TextField(blank=True, default="")

    @property
    def message_count(self):
        return self.emails.count()

class ActivityLog(TimestampedModel):
    class Action(models.TextChoices):
        # ... existing + NEW_EMAIL_RECEIVED, REOPENED, THREAD_CREATED
    thread = models.ForeignKey(Thread, on_delete=models.CASCADE, related_name="activity_logs")

Add the following functions to apps/emails/services/assignment.py:

1. `assign_thread(thread, assignee, assigned_by, note="")` — mirrors assign_email but operates on Thread:
   - Sets thread.assigned_to, assigned_by, assigned_at
   - Creates ActivityLog with thread=thread (email=None)
   - Action: ASSIGNED or REASSIGNED based on old_assignee
   - Fire-and-forget Chat notification using _send_assignment_chat (pass thread instead of email — the notifier uses .subject, .category, .priority, .pk attributes which Thread also has)
   - Fire-and-forget email notification
   - Returns updated Thread

2. `change_thread_status(thread, new_status, changed_by)` — mirrors change_status:
   - Validates against Thread.Status.values
   - Updates thread.status, saves
   - Creates ActivityLog with thread=thread, appropriate action
   - Returns updated Thread

3. `claim_thread(thread, claimed_by)` — mirrors claim_email:
   - Raises ValueError if thread.assigned_to is not None
   - Admin/staff bypasses CategoryVisibility check
   - Members need CategoryVisibility for thread.category
   - Uses assign_thread internally
   - Updates last ActivityLog from ASSIGNED to CLAIMED
   - Returns updated Thread

4. `update_thread_preview(thread)` — new function for THRD-05:
   - Queries thread.emails.order_by("-received_at").first() for latest email
   - Sets thread.last_message_at, last_sender (from_name), last_sender_address (from_address)
   - Queries thread.emails.order_by("received_at").first() for earliest email
   - Sets thread.subject from earliest email's subject
   - Queries latest COMPLETED email for triage fields
   - Sets thread.category, priority, ai_summary, ai_draft_reply from latest triaged email
   - Saves thread with update_fields for all changed fields
   - Returns None if thread has no emails (no-op)

Keep all existing email-level functions (assign_email, change_status, claim_email, auto_assign_batch) unchanged — they will be deprecated in Phase 2 but must not break existing tests now.

<task type="auto">
  <name>Task 1: Thread detail view and action endpoints</name>
  <files>apps/emails/views.py, apps/emails/urls.py</files>
  <action>
Add four new view functions to views.py for thread-level operations. Keep all existing email-level views intact (do not delete).

1. **thread_detail(request, pk)** — GET endpoint for HTMX detail panel:
   - `thread = get_object_or_404(Thread.objects.select_related("assigned_to", "assigned_by"), pk=pk)`
   - Load all emails in the thread: `emails = thread.emails.select_related("assigned_to").prefetch_related("attachments").order_by("received_at")` (oldest first)
   - For each email, sanitize body_html using nh3 (same SAFE_TAGS/SAFE_ATTRIBUTES pattern)
   - Load thread activity logs: `thread.activity_logs.select_related("user", "email").order_by("created_at")` (chronological, oldest first)
   - Build a merged timeline: combine emails and activity logs sorted by timestamp (email.received_at vs log.created_at). Each item tagged with type="message" or type="activity". This gives the interleaved view.
   - Can this user claim? Same logic as current: if unassigned, admin can always claim, members check CategoryVisibility against thread.category.
   - AI triage card data: use thread-level fields (thread.ai_summary, thread.ai_draft_reply, thread.priority, thread.category). Also include `thread.ai_suggested_assignee` if the latest email has one (check latest completed email).
   - Context: `thread`, `timeline_items` (merged list), `team_members`, `is_admin`, `can_claim`
   - Render `emails/_thread_detail.html`

2. **assign_thread_view(request, pk)** — POST endpoint (admin only):
   - Same pattern as existing `assign_email_view` but operates on Thread
   - Get thread, assignee_id from POST, optional note
   - Call `assign_thread(thread, assignee, assigned_by, note=note)` from services
   - Return: re-rendered detail panel + OOB thread card update (`<div id="thread-card-{{ thread.pk }}" hx-swap-oob="outerHTML">` rendered from `_thread_card.html`)

3. **change_thread_status_view(request, pk)** — POST endpoint:
   - Permission: admin or assigned_to user
   - Get new_status from POST
   - Call `change_thread_status(thread, new_status, changed_by)`
   - Return: re-rendered detail panel + OOB thread card update

4. **claim_thread_view(request, pk)** — POST endpoint:
   - Call `claim_thread(thread, request.user)` — handles ValueError/PermissionError
   - Return: re-rendered detail panel + OOB thread card update

5. **whitelist_sender_from_thread(request, pk)** — POST endpoint (admin only):
   - Get the thread, find the primary sender (thread.last_sender_address)
   - Same whitelist logic as existing whitelist_sender but uses thread's sender
   - Return: re-rendered detail panel with feedback banner

Add URL patterns in urls.py:
```python
path("threads/<int:pk>/detail/", views.thread_detail, name="thread_detail"),
path("threads/<int:pk>/assign/", views.assign_thread_view, name="assign_thread"),
path("threads/<int:pk>/status/", views.change_thread_status_view, name="change_thread_status"),
path("threads/<int:pk>/claim/", views.claim_thread_view, name="claim_thread"),
path("threads/<int:pk>/whitelist-sender/", views.whitelist_sender_from_thread, name="whitelist_thread_sender"),
```

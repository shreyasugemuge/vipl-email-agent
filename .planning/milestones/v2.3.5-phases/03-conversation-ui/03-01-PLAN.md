  <name>Task 1: Thread list view and URL routing</name>
  <files>apps/emails/views.py, apps/emails/urls.py</files>
  <action>
Create a `thread_list` view in views.py that replaces the current email_list as the main dashboard entry point. Keep the existing email_list view intact (do not delete it — per project rules). The thread_list view should:

1. **Base queryset**: `Thread.objects.select_related("assigned_to").exclude(status="")` ordered by `-last_message_at`.

2. **View filtering** (matching sidebar views):
   - Default view for admins: `"all_open"`. Default for members: `"mine"`.
   - `"unassigned"` — `assigned_to__isnull=True` AND status in (new, acknowledged)
   - `"mine"` — `assigned_to=user`
   - `"all_open"` — status in (new, acknowledged)
   - `"closed"` — status=closed
   - Member digit view for admin: `assigned_to_id=int(view)`

3. **Inbox filter**: `?inbox=info@vidarbhainfotech.com` filters threads to those with at least one email in that inbox: `qs = qs.filter(emails__to_inbox=inbox).distinct()`.

4. **Priority/category/status filters**: Same pattern as email_list but on Thread fields.

5. **Search**: `?q=term` searches `subject__icontains` OR `ai_summary__icontains` OR `last_sender__icontains` OR `last_sender_address__icontains`.

6. **Sidebar counts** — compute these from Thread model (not Email):
   ```python
   base_threads = Thread.objects.all()
   # Apply inbox filter to counts too
   if inbox:
       base_threads = base_threads.filter(emails__to_inbox=inbox).distinct()
   sidebar_counts = {
       "unassigned": base_threads.filter(assigned_to__isnull=True, status__in=["new", "acknowledged"]).count(),
       "mine": base_threads.filter(assigned_to=user).filter(status__in=["new", "acknowledged"]).count(),
       "all_open": base_threads.filter(status__in=["new", "acknowledged"]).count(),
       "closed": base_threads.filter(status="closed").count(),
   }
   ```

7. **Inbox list for filter pills**: Get distinct `to_inbox` values from Email model for the inbox toggle pills.

8. **Pagination**: 25 per page (same as current).

9. **Context**: Pass `threads` (page_obj), `sidebar_counts`, `current_view`, `current_inbox`, `current_priority`, `current_category`, `current_search`, `current_sort`, `inboxes` (for pills), `is_admin`, `team_members`, `total_count`, `query_params`.

10. **HTMX**: If `request.htmx`, render `emails/_thread_list_body.html`. Otherwise render `emails/thread_list.html`.

Add URL pattern: `path("", views.thread_list, name="thread_list")` — this replaces the empty path. Move old email_list to `path("legacy/", views.email_list, name="email_list")` so it remains accessible but is not the default.

Also add `path("threads/<int:pk>/detail/", views.thread_detail, name="thread_detail")` as a placeholder (the view will be created in Plan 02, for now just add the URL pattern with a comment).

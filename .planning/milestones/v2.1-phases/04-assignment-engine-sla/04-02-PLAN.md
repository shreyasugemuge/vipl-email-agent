class AssignmentRule(TimestampedModel):
    category = models.CharField(max_length=100, db_index=True)
    assignee = models.ForeignKey(settings.AUTH_USER_MODEL, on_delete=models.CASCADE, related_name="assignment_rules")
    priority_order = models.PositiveSmallIntegerField(default=0)
    is_active = models.BooleanField(default=True)
    class Meta:
        ordering = ["category", "priority_order"]
        unique_together = [("category", "assignee")]

class SLAConfig(TimestampedModel):
    priority = models.CharField(max_length=20)
    category = models.CharField(max_length=100)
    ack_hours = models.FloatField(default=1.0)
    respond_hours = models.FloatField(default=24.0)
    class Meta:
        unique_together = [("priority", "category")]

class CategoryVisibility(TimestampedModel):
    user = models.ForeignKey(settings.AUTH_USER_MODEL, on_delete=models.CASCADE, related_name="visible_categories")
    category = models.CharField(max_length=100)
    class Meta:
        unique_together = [("user", "category")]

  <action>
1. Add SLA template filters to apps/emails/templatetags/email_tags.py:
   - sla_color(deadline): returns Tailwind color string based on time remaining. If deadline is None -> "slate". If now >= deadline -> "red animate-pulse" (breached, flashing). If > 2 hours remaining -> "emerald". If 1-2 hours -> "amber". If 30min-1hr -> "orange". If < 30min -> "red".
   - sla_countdown(deadline): returns human-readable string. None -> "--". Breached -> "-Xh Ym". Remaining -> "Xh Ym" (or "Ym" if under 1 hour).
   - sla_ack_countdown(email): shorthand that calls sla_countdown on email.sla_ack_deadline.

2. Add new URL patterns to apps/emails/urls.py:
   - path("<int:pk>/claim/", views.claim_email_view, name="claim_email")
   - path("<int:pk>/accept-ai/", views.accept_ai_suggestion, name="accept_ai_suggestion")
   - path("<int:pk>/reject-ai/", views.reject_ai_suggestion, name="reject_ai_suggestion")
   - path("settings/", views.settings_view, name="settings")
   - path("settings/rules/", views.settings_rules_save, name="settings_rules_save")
   - path("settings/visibility/", views.settings_visibility_save, name="settings_visibility_save")
   - path("settings/sla/", views.settings_sla_save, name="settings_sla_save")

3. Add views to apps/emails/views.py:
   - claim_email_view(request, pk): POST, login_required. Get email, call claim_email(email, request.user). On success, return updated card + OOB detail (same pattern as assign_email_view). On ValueError/PermissionError, return 403 with message.
   - accept_ai_suggestion(request, pk): POST, login_required, admin only. Get email. Read email.ai_suggested_assignee["user_id"], look up User. Call assign_email(email, user, request.user, note="Accepted AI suggestion"). Return updated detail panel.
   - reject_ai_suggestion(request, pk): POST, login_required, admin only. Get email. Set email.ai_suggested_assignee = {} and save. Return updated detail panel.

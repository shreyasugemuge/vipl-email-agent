# Phase 4.5 Plan 01: Integration Gap Fixes Summary

Closed 4 integration gaps found during v1.0 milestone audit: body_html pipeline, chat webhook seed, quiet hours config mismatch, and accept_ai OOB card update.

## Task 1: Wire body_html through the pipeline

- Added `body_html: str = ""` field to `EmailMessage` dataclass in dtos.py
- Added `_extract_html_body()` method to `GmailPoller` that recursively extracts raw HTML from Gmail API payloads (text/html parts, base64 decoded, truncated at 50000 chars)
- Wired `body_html` into `_parse_message` so it flows from Gmail API to the DTO
- Added `body_html` to both `update_or_create` calls in pipeline.py (normal save + error-state save)
- Added `body_html` to retry EmailMessage reconstruction in `retry_failed_emails`
- Fixed stale "placeholder" comment to accurate "Send Chat notifications for newly processed emails"

## Task 2: Fix chat webhook seed, quiet hours, accept_ai OOB

- Changed quiet_hours_start/end in seed migration from int "20"/"8" to str "20:00"/"08:00"
- Added `chat_webhook_url` entry to seed migration (empty string, str type, notifications category)
- Added normalization in `ChatNotifier._is_quiet_hours()` to handle both int ("20") and "HH:MM" ("20:00") formats before parsing
- Added `import os` and env var fallback (`GOOGLE_CHAT_WEBHOOK_URL`) for webhook_url in both `assign_email` and `auto_assign_batch` functions
- Updated `accept_ai_suggestion` to return OOB card+detail update (matching `assign_email_view` pattern)
- Updated `reject_ai_suggestion` to return OOB card+detail update (AI badge clears immediately)
- Updated seed config test to match new HH:MM format and verify chat_webhook_url presence

  <action>
    Create apps/emails/services/pipeline.py -- the orchestrator module. This is the ONLY service module that imports Django ORM. Per locked decision: "GmailPoller returns EmailMessage, a save step maps it to the Email model."

    Functions:
    1. save_email_to_db(email_msg: EmailMessage, triage: TriageResult) -> Email:
       - Use Email.objects.update_or_create(message_id=email_msg.message_id, defaults={...})
       - Map ALL EmailMessage fields to Email model fields (from_address, from_name, to_inbox, subject, body, received_at, gmail_id, gmail_thread_id, gmail_link)
       - Map ALL TriageResult fields to Email model fields (category, priority, ai_summary=triage.summary, ai_draft_reply=triage.draft_reply, language, is_spam, ai_reasoning=triage.reasoning, ai_model_used=triage.model_used, ai_tags=triage.tags, ai_suggested_assignee=triage.suggested_assignee, ai_input_tokens, ai_output_tokens, spam_score)
       - Set processing_status="completed" on successful save
       - Save AttachmentMetadata for each item in email_msg.attachment_details
       - Return the Email instance

    2. process_single_email(email_msg, ai_processor, gmail_poller, spam_filter_fn, chat_enabled) -> Optional[Email]:
       - Check spam first: spam_result = spam_filter_fn(email_msg)
       - If spam: save_email_to_db with spam result, mark_processed, return
       - If AI enabled: triage = ai_processor.process(email_msg, gmail_poller=gmail_poller)
       - If AI disabled: triage = AIProcessor._fallback_result("AI disabled")
       - try: email_obj = save_email_to_db(email_msg, triage)
       - THEN: gmail_poller.mark_processed(email_msg)  # LABEL-AFTER-PERSIST
       - except: set processing_status="failed", last_error, save, re-raise or return None

    3. process_poll_cycle(gmail_poller, ai_processor, chat_notifier, state_manager):
       - Call close_old_connections() first (Django DB pitfall from RESEARCH.md)
       - Read SystemConfig: ai_triage_enabled, chat_notifications_enabled, monitored_inboxes, max_consecutive_failures
       - Check circuit breaker: if state_manager.consecutive_failures >= max, log critical and return
       - try: poll gmail_poller.poll_all(inboxes)
       - For each email: skip if Email.objects.filter(message_id=...).exists() (dedup)
       - Call process_single_email for each
       - If chat_enabled and processed_items: call chat_notifier.notify_new_emails(items) (placeholder -- Chat notifier built in Plan 03)
       - state_manager.reset_failures() on success
       - except: state_manager.record_failure(), log error

    4. retry_failed_emails(ai_processor, gmail_poller):
       - Call close_old_connections()
       - Query Email.objects.filter(processing_status="failed", retry_count__lt=3).order_by("created_at")[:10]
       - For each: increment retry_count, attempt re-triage, update fields on success (processing_status="completed"), mark exhausted after 3rd failure
       - Per RESEARCH.md pattern

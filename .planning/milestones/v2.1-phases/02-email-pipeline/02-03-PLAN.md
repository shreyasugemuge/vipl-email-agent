  <behavior>
    ChatNotifier:
    - ChatNotifier.__init__ takes webhook_url
    - notify_new_emails(emails) posts a summary card to webhook (batch, not per-email)
    - _is_quiet_hours() checks SystemConfig quiet_hours_start/end against current IST time
    - Notifications suppressed during quiet hours (log but don't post)
    - Uses httpx.post with 10s timeout
    - Returns silently on webhook error (log warning, don't crash pipeline)

    Scheduler:
    - `python manage.py run_scheduler` starts APScheduler BlockingScheduler
    - Heartbeat job runs every 1 minute, writes timestamp to SystemConfig key "scheduler_heartbeat"
    - Poll job runs every 5 minutes (reads interval from SystemConfig), calls process_poll_cycle
    - Retry job runs every 30 minutes, calls retry_failed_emails
    - All jobs call close_old_connections() at start
    - All jobs have max_instances=1, coalesce=True
    - Signal handlers for SIGTERM/SIGINT shut down scheduler gracefully
    - Services (GmailPoller, AIProcessor, ChatNotifier, StateManager) initialized once in handle()

    Health endpoint:
    - Existing health_check adds scheduler_status field
    - Reads SystemConfig.get("scheduler_heartbeat") timestamp
    - If heartbeat is < 5 minutes old: "running"
    - If heartbeat is >= 5 minutes old or missing: "stale" (status becomes "degraded")
  </behavior>

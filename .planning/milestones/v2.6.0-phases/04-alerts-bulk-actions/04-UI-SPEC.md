# Phase 4 — UI Design Contract

> Visual and interaction contract for alerts, bulk actions, and AI corrections digest. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Django templates + Tailwind CSS v4 pre-built) |
| Preset | not applicable |
| Component library | none (HTMX 2.0 + vanilla JS) |
| Icon library | inline SVG (heroicons style, already established) |
| Font | Plus Jakarta Sans (Google Fonts, weights 300-800 loaded) |

---

## Visual Focal Point

When no selection is active, the digest card is the primary visual anchor. When selection is active, the floating bar is the primary visual anchor.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (Tailwind `gap-1`, `px-1`) |
| sm | 8px | Compact element spacing (Tailwind `gap-2`, `px-2`, `py-2`) |
| md | 16px | Default element spacing, card padding (Tailwind `gap-4`, `px-4`) |
| lg | 24px | Section padding, floating bar internal padding (Tailwind `px-6`) |
| xl | 32px | Layout gaps (Tailwind `gap-8`) |
| 2xl | 48px | Major section breaks (Tailwind `py-12`) |
| 3xl | 64px | Page-level spacing (Tailwind `py-16`) |

Exceptions: Floating action bar uses 12px vertical padding (`py-3`) to stay compact at viewport bottom. Thread card checkbox uses 44px minimum touch target on mobile per WCAG.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | 11px (`text-[11px]`) | 600 (semibold) | 1.4 |
| Heading | 16px (`text-base`) | 600 (semibold) | 1.2 |
| Caption | 10px (`text-[10px]`) | 600 (semibold) | 1.3 |

Source: Matches existing thread card typography. Body at 14px for floating bar labels, digest card text. Caption at 10px for stat badges, counts, pattern headings. Two weights only: 400 (regular) for body text, 600 (semibold) for emphasis, labels, headings, and captions.

## Component Inventory

### 1. Floating Bulk Action Bar

**Location:** Fixed to viewport bottom, centered horizontally.
**Visibility:** Hidden by default. Slides up when >= 1 thread checkbox is checked. Slides down on clear or after action.
**Layout:** Horizontal flex: `[N selected] [Assign to: dropdown] [Assign Threads button] [|] [Mark Irrelevant button] [Clear button]`

| Property | Value |
|----------|-------|
| Position | `fixed bottom-4 left-1/2 -translate-x-1/2` |
| Width | `max-w-lg w-[calc(100%-2rem)]` (responsive, never wider than 512px) |
| Background | `bg-white` |
| Border | `border border-slate-200 rounded-2xl` |
| Shadow | `shadow-xl shadow-black/10` |
| Padding | `px-4 py-3` |
| Z-index | `z-50` |
| Transition | `transition-all duration-300 ease-out` |
| Hidden state | `translate-y-full opacity-0 pointer-events-none` |
| Visible state | `translate-y-0 opacity-100` |

**Internal elements:**

| Element | Spec |
|---------|------|
| Selected count | `text-sm font-semibold text-slate-700` -- e.g., "3 selected" |
| Assignee dropdown | `<select>` with `text-sm border border-slate-200 rounded-lg px-2 py-2` |
| Assign Threads button | `bg-primary-500 hover:bg-primary-600 text-white text-sm font-semibold px-3 py-2 rounded-lg` |
| Divider | `w-px h-6 bg-slate-200 mx-1` (vertical line) |
| Mark Irrelevant button | `bg-red-50 hover:bg-red-100 text-red-700 text-sm font-semibold px-3 py-2 rounded-lg border border-red-200` |
| Clear selection | `text-slate-400 hover:text-slate-600 text-sm underline` -- text "Clear" |

**Mobile (< 640px):** Stack vertically. Selected count on top row, buttons on bottom row. `flex-col gap-2`.

### 2. Thread Card Checkbox

**Location:** Left side of thread card, before the card content.
**Desktop:** `opacity-0 group-hover:opacity-100` -- appears on card hover.
**Mobile (< 640px):** `opacity-100` -- always visible.
**Checked state:** Card gets `card-selected` class (existing: pink gradient background + pink left border).

| Property | Value |
|----------|-------|
| Size | `w-4 h-4` (16px, but touch target wrapping `min-w-[44px] min-h-[44px]` on mobile) |
| Position | `absolute left-1 top-1/2 -translate-y-1/2` inside card (card needs `relative group`) |
| Color when checked | `accent-primary-500` |
| Transition | `opacity transition-opacity duration-150` |
| Interaction | `onclick` stops propagation (prevent card click opening detail) |

### 3. Select All Checkbox

**Location:** Thread list header row, above the thread cards (inside `_thread_list_body.html` or the stat card row area).

| Property | Value |
|----------|-------|
| Label | `text-[11px] font-semibold text-slate-500` -- "Select all" |
| Checkbox | `w-3.5 h-3.5 accent-primary-500` |
| Position | Left-aligned in the search/count bar area |
| Visibility | Only visible to users with `can_assign` permission |

### 4. Unassigned Count Badge (Sidebar)

**Location:** Existing sidebar "Unassigned" pill count badge. Extends current pattern with threshold-based coloring.

| Count Range | Background | Text Color | Matches |
|-------------|------------|------------|---------|
| 0 | `bg-slate-100` | `text-slate-400` | Neutral |
| 1-2 | `bg-green-100` | `text-green-700` | Safe |
| 3-4 | `bg-amber-100` | `text-amber-700` | Warning |
| 5+ | `bg-red-100` | `text-red-700` | Alert threshold |

Badge shape: `inline-flex items-center justify-center px-2 py-0.5 rounded-full text-[10px] font-semibold min-w-[20px] text-center` (matches existing sidebar badge pattern).

### 5. AI Corrections Digest Card

**Location:** Above thread list body, below stat cards, inside `thread_list.html`. Visible only to gatekeeper + admin.
**State:** Collapsible. Expanded by default. Collapsed state persisted in `localStorage`.

| Property | Value |
|----------|-------|
| Wrapper | `mx-3 mb-2 rounded-xl border border-blue-200 bg-blue-50 overflow-hidden` |
| Header | `flex items-center justify-between px-4 py-2.5 cursor-pointer` |
| Header icon | Lightbulb SVG, `w-4 h-4 text-blue-500` |
| Header title | `text-[11px] font-semibold text-blue-700` -- "AI Corrections (7 days)" |
| Header total | `text-[11px] font-semibold text-blue-500` -- e.g., "12 corrections" |
| Collapse chevron | `w-4 h-4 text-blue-400 transition-transform duration-200` -- rotates 180deg when collapsed |
| Body | `px-4 pb-3` |
| Body transition | `max-height` transition with `overflow-hidden`, 200ms ease |

**Body content layout:**

| Section | Spec |
|---------|------|
| Correction counts row | Horizontal flex with 3 mini-stats: `[icon] N category [icon] N priority [icon] N spam` |
| Count labels | `text-[11px] font-semibold text-slate-600` |
| Count values | `text-sm font-semibold` in their respective colors (amber/red/orange per color table) |
| Divider | `border-t border-blue-100 my-2` |
| Patterns heading | `text-[10px] font-semibold text-blue-600 uppercase tracking-wider mb-1` -- "Top Patterns" |
| Pattern items | `text-[11px] text-slate-600` -- e.g., "General -> Govt (5x)" |
| Pattern list | `space-y-0.5`, max 5 items |
| Empty state | `text-[11px] text-blue-400 italic` -- "No corrections in the last 7 days" |

### 6. Undo Toast

**Location:** Existing toast container (`#toast-container`), top-right.
**Trigger:** `HX-Trigger: showUndoToast` response header after bulk action.
**Duration:** 10 seconds visible, then auto-dismiss.

| Property | Value |
|----------|-------|
| Layout | `flex items-center gap-3 px-4 py-3 bg-white rounded-xl shadow-lg border border-slate-200 max-w-sm` (matches existing toast) |
| Message | `text-sm text-slate-700` -- e.g., "Assigned 3 threads to Shreyas" |
| Undo button | `text-sm font-semibold text-blue-600 hover:text-blue-800 cursor-pointer` -- "Undo" |
| Close button | Same as existing toast close (X icon, 44px touch target) |
| Animation in | `toast-in 0.3s ease-out` (existing keyframe) |
| Animation out | `toast-out 0.3s ease-in forwards` (existing keyframe) |
| Auto-dismiss | 10 seconds (vs 4 seconds for regular toasts) |

### 7. Bulk Mark-Irrelevant Reason Input

**Trigger:** Clicking "Mark Irrelevant" in floating bar opens an inline text input within the bar.
**Behavior:** Bar expands to show: `[Reason input] [Mark as Irrelevant button] [Cancel]`. Replaces the normal bar content.

| Property | Value |
|----------|-------|
| Input | `text-sm border border-red-200 rounded-lg px-3 py-2 flex-1 placeholder-slate-400` |
| Placeholder | "Why are these irrelevant?" |
| Confirm button | `bg-red-600 hover:bg-red-700 text-white text-sm font-semibold px-3 py-2 rounded-lg` -- "Mark as Irrelevant" |
| Cancel | `text-sm text-slate-400 hover:text-slate-600 underline` -- "Cancel" |
| Validation | Input must be non-empty to enable confirm (disabled state: `opacity-50 cursor-not-allowed`) |

## Interaction Contracts

### Bulk Selection Flow

1. User hovers over thread card -> checkbox fades in (desktop) or is always visible (mobile)
2. User clicks checkbox -> card gets `card-selected` class, floating bar slides up from bottom
3. User checks more cards -> selected count updates in floating bar, hidden `thread_ids` inputs update
4. User clicks "Select all" -> all visible thread checkboxes checked, count updates
5. User picks assignee from dropdown + clicks "Assign Threads" -> HTMX POST to `/emails/threads/bulk-assign/`
6. Server returns updated `#thread-list-body` via `hx-swap="innerHTML"` + `HX-Trigger: showUndoToast`
7. Floating bar slides down (selection cleared by DOM replacement)
8. Undo toast appears top-right with 10-second window
9. User clicks "Undo" -> HTMX POST to `/emails/threads/bulk-undo/` with previous state payload
10. Thread list refreshes again, toast dismissed

### Mark Irrelevant Flow

1. User selects threads, clicks "Mark Irrelevant" in floating bar
2. Bar content swaps to reason input mode (same bar, different content)
3. User types reason, clicks "Mark as Irrelevant" -> HTMX POST to `/emails/threads/bulk-mark-irrelevant/`
4. Same response pattern as bulk assign (swap + undo toast)
5. User clicks "Cancel" -> bar reverts to normal mode, selection preserved

### Digest Card Interaction

1. Page loads with triage queue view -> digest card rendered above thread list (gatekeeper/admin only)
2. If `localStorage.getItem('digest_collapsed') === 'true'` -> render collapsed
3. Click header -> toggle collapse, save to `localStorage`
4. No HTMX interaction -- fully server-rendered on page load, JS only for collapse toggle

### Unassigned Badge Color

1. Server renders badge with threshold-based color class in template
2. No client-side logic -- color is computed server-side based on count value
3. Badge updates when sidebar is re-rendered via HTMX navigation

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Floating bar announced | `role="toolbar" aria-label="Bulk actions for {N} selected threads"` |
| Checkbox labels | `aria-label="Select thread: {subject}"` on each checkbox |
| Select all | `aria-label="Select all visible threads"` |
| Undo toast | `role="status" aria-live="polite"` (existing toast container) |
| Digest collapse | `aria-expanded="true/false"` on header, `aria-controls="digest-body"` |
| Keyboard: Escape | Clears selection and hides floating bar |
| Keyboard: checkbox | Space/Enter toggles checkbox (native behavior) |
| Focus management | After bulk action, focus returns to thread list area |
| Color contrast | All badge text meets 4.5:1 ratio (green-700 on green-100, amber-700 on amber-100, red-700 on red-100) |

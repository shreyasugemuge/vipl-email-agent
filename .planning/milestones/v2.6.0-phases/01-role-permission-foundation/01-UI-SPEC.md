# Phase 1 — UI Design Contract

> Visual and interaction contract for the Triage Lead role UI elements. Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Django templates + Tailwind CSS v4) |
| Preset | not applicable |
| Component library | none (server-rendered HTML + HTMX 2.0) |
| Icon library | Inline SVG (heroicons style, already used throughout) |
| Font | Plus Jakarta Sans (Google Fonts, weights 300-800 loaded) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |

| 3xl | 64px | Page-level spacing |

| Exceptions | Justification |
|------------|---------------|

## Visual Hierarchy

**Primary visual anchor for Triage Lead's main view:** The thread list card stack is the dominant element. The sidebar category scope pills orient the Triage Lead to their filtered domain. Unassigned thread count in the "Triage Queue" stat card is the single most important number on the page — it tells the Triage Lead how much work remains.

## Component Inventory

### New UI Elements (Phase 1)

#### 1. Triage Lead Role Badge
- **Location:** Team page `_user_row.html` (self-view), sidebar user section
- **Pattern:** Matches existing admin badge: `<span class="px-2 py-0.5 rounded text-[9px] font-bold uppercase tracking-wide bg-blue-50 text-blue-600">Triage Lead</span>`
- **Interaction:** Static display, no click action

#### 2. Role Dropdown (3 options)
- **Location:** Team page `_user_row.html` (other users, admin only)
- **Current:** 2 hardcoded `<option>` tags (Admin, Member)
- **New:** Add `<option value="triage_lead">Triage Lead</option>` between Admin and Member
- **Interaction:** HTMX `hx-post` on change (existing pattern), server responds with updated row partial
- **Visibility:** Only shown when `user.is_admin_only` (admin cannot demote themselves, triage lead cannot change roles)

#### 3. Dev Login Triage Lead Button
- **Location:** `dev_login.html`, third form after Admin and Member
- **Pattern:** Same card style as existing buttons
- **Icon:** Shield with checkmark or clipboard-list SVG (blue-100 bg, blue-600 icon)
- **Interaction:** POST form with `role=triage_lead`

#### 4. Sidebar Category Scope Pills
- **Location:** Sidebar, below "Main" nav section, above "System" section (only for triage_lead users)
- **Pattern:** Small rounded pills showing each assigned category name
- **Layout:** `<div class="px-3 flex flex-wrap gap-1">` with each pill as `<span class="px-2 py-0.5 rounded text-[9px] font-semibold bg-white/[0.08] text-slate-400 border border-white/[0.1]">{category}</span>`
- **Empty state:** Single line italic text (no pills)
- **Interaction:** Static display, no click action

#### 5. Settings Read-Only Banner
- **Location:** Top of settings page, below page title
- **Pattern:** Info banner: `bg-blue-50 border border-blue-200 rounded-xl px-4 py-3`
- **Icon:** Info circle SVG (blue-500), left-aligned
- **Text:** 11px, font-semibold, text-blue-700
- **Interaction:** Static, not dismissible. All form inputs below have `disabled` attribute. Save buttons hidden or disabled with `opacity-50 cursor-not-allowed`.

#### 6. Welcome Banner (Triage Lead variant)
- **Location:** `email_list.html` welcome banner section
- **Pattern:** Same gradient banner as existing (`bg-gradient-to-r from-primary-50 to-primary-100`)
- **Logic:** Add `{% elif user.is_triage_lead %}` branch between admin and member conditions
- **Interaction:** Same dismiss behavior (session + permanent via localStorage)

### Modified UI Elements (Phase 1)

#### 7. Sidebar Navigation Gates
- **Current:** Single `{% if user.is_staff or user.is_admin_role %}` wrapping Settings, Reports, Team, Inspector
- **New:** Per-link visibility checks:
  - Reports: `{% if user.can_assign %}` (admin + triage_lead)
  - Team: `{% if user.can_approve_users %}` (admin + triage_lead)
  - Settings: `{% if user.can_triage %}` (admin + triage_lead, read-only for triage_lead)
  - Dev Inspector: `{% if user.can_triage %}` (admin + triage_lead, view-only for triage_lead)
- **Visual:** No change to link appearance. Same 13px font, same icons, same hover states.

#### 8. Sidebar Role Label
- **Current:** `<div class="text-[9px] text-slate-600 font-medium">{{ user.role|default:"Admin" }}</div>`
- **New:** Same markup, `get_role_display` will return "Triage Lead" for the new role. Display: "Triage Lead" in slate-600, 9px.

## Interaction States

### Role Change (Team Page)

| State | Visual |
|-------|--------|
| Default | Role dropdown shows current role selected |
| Loading (HTMX request) | Row opacity 0.7 (existing `htmx-request` style) |
| Success | Row re-rendered via HTMX partial swap with new role |
| Error | Toast notification with error message |

### Settings Read-Only (Triage Lead)

| State | Visual |
|-------|--------|
| Page load | Blue info banner visible at top, all inputs disabled, save buttons hidden |
| Tab navigation | Tabs still clickable (view-only applies to all tabs equally) |
| Form interaction | Disabled inputs show `cursor-not-allowed`, no hover/focus ring |

### Category Scope Filtering

| State | Visual |
|-------|--------|
| Triage Lead with categories | Thread list shows only matching threads, sidebar counts reflect filtered totals |
| Triage Lead with no categories | Empty state illustration, guidance text, no thread cards |
| Sidebar pills | Static category pills visible below nav |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Role badge screen reader | Badge text is visible, no extra aria needed |
| Disabled settings inputs | `disabled` attribute provides native screen reader announcement |
| Settings read-only banner | `role="status"` on the info banner div |
| Category pills | `aria-label="Your assigned categories"` on the pills container |
| Role dropdown | Existing `<select>` is natively accessible |
| Dev login new button | Same focusable form pattern as existing buttons |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Mobile (<768px) | Sidebar category pills visible when sidebar is open (slide-out). Settings read-only banner stacks vertically. Role dropdown in team page may be hidden on mobile (existing pattern hides some columns). |
| Desktop (>=768px) | All elements visible as specified above. |

No new responsive breakpoints needed. All new elements follow existing responsive patterns.

       - No quiet hours check (alerts fire regardless)
       - Build breakdown text: sort categories by count descending, format as "3 Sales, 2 Support, 2 General"
       - triage_url = f"{self._tracker_url}/emails/?view=unassigned"
       - Card header: branded_header with title "Warning: {count} unassigned threads (threshold: {threshold})", subtitle = breakdown_text or "Triage queue needs attention"

       - Return self._post(payload)

    3. Add `_check_unassigned_alert()` function in run_scheduler.py (module-level, like `_heartbeat_job`):
       - Call `close_old_connections()` first
       - Read threshold: `SystemConfig.objects.filter(key="unassigned_alert_threshold").first()` -- if not found or value is "0", return (alerts disabled)
       - Parse threshold as int
       - Count unassigned: `Thread.objects.filter(assigned_to__isnull=True, status__in=["new", "acknowledged"]).count()`
       - Read `_unassigned_was_below_threshold` from SystemConfig (default "true")
       - Rising-edge logic:
         - If count >= threshold AND was_below == "true":
           - Check cooldown: read `last_unassigned_alert_at` and `unassigned_alert_cooldown_minutes` from SystemConfig, compare with timezone.now(). If within cooldown, return.
           - Get webhook_url from SystemConfig("chat_webhook_url") or os.environ.get("GOOGLE_CHAT_WEBHOOK_URL")
           - If webhook_url: build category breakdown using `Thread.objects.filter(assigned_to__isnull=True, status__in=["new", "acknowledged"]).values("category").annotate(c=Count("pk"))`, instantiate ChatNotifier, call `notify_unassigned_alert(count, threshold, breakdown)`
           - Update `last_unassigned_alert_at` to `timezone.now().isoformat()` via `SystemConfig.objects.update_or_create`
           - Set `_unassigned_was_below_threshold` to "false" via `SystemConfig.objects.update_or_create`
         - If count >= threshold AND was_below != "true": do nothing (already above, no re-alert)
         - If count < threshold AND was_below != "true": set `_unassigned_was_below_threshold` to "true" (reset for next rising edge)
       - Wrap entire function in try/except with `logger.error(f"Unassigned alert check failed: {e}")`

    4. Modify `_heartbeat_job()` to call `_check_unassigned_alert()` at the end (after the existing SystemConfig heartbeat write). Wrap in separate try/except so heartbeat write still succeeds even if alert check fails.

    5. Add import `from django.db.models import Count` at top of run_scheduler.py. Add import for Thread model: `from apps.emails.models import Thread`.

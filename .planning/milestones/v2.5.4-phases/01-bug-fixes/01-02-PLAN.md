<task type="auto">
  <name>Task 1: Fix welcome banner double-show on OAuth login</name>
  <files>templates/emails/email_list.html</files>
  <action>
The welcome banner in `email_list.html` (line ~258-285) shows twice because:
1. OAuth login redirects: login page -> Google -> callback -> dashboard
2. The dashboard page may load twice in quick succession (HTMX partial + full page)
3. sessionStorage flag is set on dismiss, but the banner auto-shows before dismiss happens

Fix the JS IIFE that controls the welcome banner:
1. Immediately set `sessionStorage.setItem('vipl_welcome_shown', '1')` BEFORE removing `hidden` class
2. Check for `vipl_welcome_shown` (shown-this-session) in addition to existing `vipl_welcome_dismissed` and `vipl_welcome_permanent`
3. This ensures: first load sets "shown", second load (if any) sees "shown" and skips

Updated JS should be:
```javascript
(function() {
    var banner = document.getElementById('welcome-banner');
    if (!banner) return;
    var dismissed = sessionStorage.getItem('vipl_welcome_dismissed');
    var permanent = localStorage.getItem('vipl_welcome_permanent');
    var alreadyShown = sessionStorage.getItem('vipl_welcome_shown');
    if (!dismissed && !permanent && !alreadyShown) {
        sessionStorage.setItem('vipl_welcome_shown', '1');
        banner.classList.remove('hidden');
        var autoTimer = setTimeout(function() {
            banner.style.animation = 'toast-out 0.3s ease forwards';
            setTimeout(function() { banner.classList.add('hidden'); }, 300);
        }, 8000);
        banner._autoTimer = autoTimer;
    }
})();
```

This is a one-line addition (`alreadyShown` check + set) that prevents the banner from showing a second time even if the page reloads within the same session.

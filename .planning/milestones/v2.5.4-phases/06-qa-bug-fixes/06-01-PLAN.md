**QA-02: Search resets sidebar view filter**

The hidden input `<input type="hidden" name="view-hidden" value="{{ current_view }}">` at line 207 of `thread_list.html` only gets its value from the initial page render. When sidebar nav changes the view via HTMX push-URL, the hidden input value is never updated.

The search input at line 278 includes `[name='view-hidden']` via `hx-include`, so it sends the stale initial value.

Fix: In the `<script>` block of `thread_list.html` (inside `{% block extra_js %}`), add a listener for `htmx:pushedIntoHistory` that syncs the hidden input values from the current URL. Add this code after the existing `updateActiveStates()` call (around line 369):

```javascript
/* Sync hidden filter inputs when URL changes via HTMX navigation */
document.body.addEventListener('htmx:pushedIntoHistory', function() {
    var params = new URLSearchParams(window.location.search);
    var viewInput = document.querySelector('input[name="view-hidden"]');
    var inboxInput = document.querySelector('input[name="inbox-hidden"]');
    if (viewInput) viewInput.value = params.get('view') || 'all_open';
    if (inboxInput) inboxInput.value = params.get('inbox') || '';
});
```

Also add the same sync on `popstate` (browser back/forward) by extending the listener:
```javascript
window.addEventListener('popstate', function() {
    var params = new URLSearchParams(window.location.search);
    var viewInput = document.querySelector('input[name="view-hidden"]');
    var inboxInput = document.querySelector('input[name="inbox-hidden"]');
    if (viewInput) viewInput.value = params.get('view') || 'all_open';
    if (inboxInput) inboxInput.value = params.get('inbox') || '';
});
```


On mobile (<1024px), `hidden` sets `display:none` on the parent. Even though the `htmx:afterSwap` handler at line 353 removes `translate-x-full` from the inner `#thread-detail-panel`, the parent is still `display:none` so nothing shows.

Fix: Change the parent div's classes so it doesn't use `hidden` for mobile. Instead, use a fixed/overlay approach for mobile and a flex layout for desktop:

Replace line 312:
```html
<div class="hidden lg:flex flex-1 min-w-0">
```
With:
```html
<div id="thread-detail-wrapper" class="contents lg:flex lg:flex-1 lg:min-w-0">
```

Wait -- `contents` makes the div invisible to layout but keeps children in flow. That won't work well either. Better approach: just remove `hidden` and make the right panel a fixed overlay on mobile that starts off-screen:

**QA-04: Escape key doesn't close detail panel**

The keydown handler at line 410-411 only calls `closeContextMenu()`:
```javascript
if (e.key === 'Escape') { closeContextMenu(); return; }
```

Fix: Before the `return`, also close the detail panel if it's open. Change line 411 to:
```javascript
if (e.key === 'Escape') {
    closeContextMenu();
    var panel = document.getElementById('thread-detail-panel');
    if (panel && !panel.classList.contains('translate-x-full')) {
        closeThreadDetail();
    }
    return;
}
```

This checks if the detail panel is visible (doesn't have `translate-x-full`) before calling `closeThreadDetail()`. On desktop (lg+), the panel is always visible via `lg:translate-x-0` which overrides, so `translate-x-full` won't be in classList on desktop after content loads. To be safe, also check `window.innerWidth < 1024` so Escape only closes on mobile where the panel is an overlay:

```javascript
if (e.key === 'Escape') {
    closeContextMenu();
    if (window.innerWidth < 1024) {
        var panel = document.getElementById('thread-detail-panel');
        if (panel && !panel.classList.contains('translate-x-full')) {
            closeThreadDetail();
        }
    }
    return;
}
```

Actually, re-reading QA-04 requirement: "Pressing Escape closes the thread detail panel" -- this should work on all viewports. On desktop, closing means clearing the panel content and showing the placeholder. Update `closeThreadDetail()` to handle both:

```javascript
function closeThreadDetail() {
    var panel = document.getElementById('thread-detail-panel');
    if (!panel) return;
    if (window.innerWidth < 1024) {
        // Mobile: slide panel off-screen
        panel.classList.add('translate-x-full');
        var overlay = document.getElementById('thread-detail-overlay');
        if (overlay) overlay.classList.add('hidden');
    } else {
        // Desktop: restore placeholder content
        panel.innerHTML = '<div class="flex flex-col items-center justify-center w-full h-full">' +
            '<div class="w-16 h-16 rounded-2xl bg-slate-100 flex items-center justify-center mb-4">' +
            '<svg class="w-7 h-7 text-slate-300" fill="none" stroke="currentColor" viewBox="0 0 24 24">' +
            '<path stroke-linecap="round" stroke-linejoin="round" stroke-width="1.5" d="M3 8l7.89 5.26a2 2 0 002.22 0L21 8M5 19h14a2 2 0 002-2V7a2 2 0 00-2-2H5a2 2 0 00-2 2v10a2 2 0 002 2z"/>' +
            '</svg></div>' +
            '<span class="text-[13px] font-semibold text-slate-400">Select a thread to view</span>' +
            '<span class="text-[11px] text-slate-300 mt-1">Click any conversation on the left</span></div>';
    }
}
```

```javascript
if (e.key === 'Escape') {
```

<objective>
Fix two pipeline gaps found in the v2.5.0 milestone audit:

1. (P0) `update_thread_preview()` unconditionally overwrites `thread.category` and `thread.priority` from the latest AI triage, ignoring the `category_overridden` and `priority_overridden` flags set by Phase 5 inline edit views. User corrections are silently lost when the next email arrives.

2. (P1) `auto_assign_confidence_threshold` config key implies a numeric value but actually requires an exact confidence tier string like "HIGH". Rename to `auto_assign_confidence_tier` for clarity.

Purpose: Close the last unsatisfied requirement (INTEL-11) and fix misleading config naming.
Output: Patched assignment.py, pipeline.py, data migration, and comprehensive tests.
</objective>

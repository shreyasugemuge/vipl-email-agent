    - Test: POST reject_thread_suggestion creates AssignmentFeedback(action=REJECTED)

  <action>
1. In `views.py`, add two new views:

**accept_thread_suggestion(request, pk):**
- Require POST + admin role (same pattern as existing accept_ai_suggestion)
- Get thread with select_related("assigned_to")
- Get latest email in thread to read ai_suggested_assignee JSON
- Resolve suggested user from JSON `user_id` field (get_object_or_404 User)
- Assign thread: `thread.assigned_to = assignee`, `thread.assigned_by = request.user`, `thread.assigned_at = timezone.now()`, `thread.is_auto_assigned = False`
- Save thread with update_fields
- Create AssignmentFeedback: action=ACCEPTED, suggested_user=assignee, actual_user=assignee, confidence_at_time=thread.ai_confidence, user_who_acted=request.user
- Create ActivityLog: thread=thread, user=request.user, action=ASSIGNED, detail="Accepted AI suggestion"
- Return rendered thread detail partial (same pattern as existing views)

**reject_thread_suggestion(request, pk):**
- Require POST + admin role
- Get thread
- Record old assignee (if any -- for auto-assigned reject case)
- Unassign: `thread.assigned_to = None`, `thread.assigned_by = None`, `thread.assigned_at = None`, `thread.is_auto_assigned = False`
- Save thread
- Get latest email for suggested user info
- Create AssignmentFeedback: action=REJECTED, suggested_user=(from email's ai_suggested_assignee), actual_user=None, confidence_at_time=thread.ai_confidence, user_who_acted=request.user
- Clear the ai_suggested_assignee on the latest email (set to {}) so suggestion bar disappears
- Return rendered thread detail partial

2. In `urls.py`, add URL patterns (in the thread-level section):
```python
path("threads/<int:pk>/accept-suggestion/", views.accept_thread_suggestion, name="accept_thread_suggestion"),
path("threads/<int:pk>/reject-suggestion/", views.reject_thread_suggestion, name="reject_thread_suggestion"),
```

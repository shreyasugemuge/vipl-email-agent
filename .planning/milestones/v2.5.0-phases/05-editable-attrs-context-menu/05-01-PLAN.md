  <behavior>
    - POST /emails/threads/1/edit-category/ with category=Sales Lead -> thread.category becomes "Sales Lead", category_overridden=True, ActivityLog created with action=CATEGORY_CHANGED
    - POST /emails/threads/1/edit-category/ with category=__custom__&custom_category=My Custom -> thread.category becomes "My Custom"
    - POST /emails/threads/1/edit-priority/ with priority=HIGH -> thread.priority becomes "HIGH", priority_overridden=True, ActivityLog created with action=PRIORITY_CHANGED
    - POST /emails/threads/1/edit-priority/ with invalid priority -> returns 400
    - POST /emails/threads/1/edit-status/ with new_status=acknowledged -> thread.status changes, ActivityLog created
    - All three endpoints return detail HTML + OOB card HTML (same pattern as change_thread_status_view)
    - Non-POST requests return 405
    - Non-logged-in users get redirected
    - Permission: any logged-in user can edit category/priority; status follows existing rules (admin or assigned user)
  </behavior>
  <action>
    Create three view functions following the exact pattern of `edit_ai_summary` and `change_thread_status_view`:

    **edit_category(request, pk):**
    - `@login_required` + `@require_POST`
    - Read `category` from POST. If value is `__custom__`, read `custom_category` instead.
    - Validate: custom category must be non-empty and <= 100 chars. Standard categories checked against VALID_CATEGORIES.
    - Save: `thread.category = new_cat`, `thread.category_overridden = True`, `thread.save(update_fields=["category", "category_overridden"])`
    - ActivityLog: action=CATEGORY_CHANGED, old_value, new_value
    - Return: re-rendered `_thread_detail.html` + OOB `_thread_card.html`

    **edit_priority(request, pk):**
    - `@login_required` + `@require_POST`
    - Validate against VALID_PRIORITIES
    - Save: `thread.priority = new_pri`, `thread.priority_overridden = True`, `thread.save(update_fields=["priority", "priority_overridden"])`
    - ActivityLog: action=PRIORITY_CHANGED, old_value, new_value
    - Return: re-rendered detail + OOB card

    **edit_status(request, pk):**
    - `@login_required` + `@require_POST`
    - Permission: admin or assigned_to user (same as change_thread_status_view)
    - Reuse `_change_thread_status(thread, new_status, user)` from assignment service
    - Return: re-rendered detail + OOB card

    **Wave 0: Create test file** `apps/emails/tests/test_spam_feedback.py` with all tests listed above. Use existing conftest fixtures (user, admin_user, thread factory patterns from other test files). Tests should fail initially (RED).

    **Then implement:**

    1. **Views in `apps/emails/views.py`** -- add 3 new views following the `whitelist_sender_from_thread` pattern exactly:

       `mark_spam(request, pk)`:
       - `@login_required` + `@require_POST` (NO admin check -- all users can mark spam per user decision)
       - Get thread via `get_object_or_404(Thread, pk=pk)`
       - Get original verdict from thread emails: `any(e.is_spam for e in thread.emails.all())`
       - Create `SpamFeedback(user=request.user, thread=thread, original_verdict=original, user_verdict=True)`
       - Update all thread emails: `thread.emails.update(is_spam=True)`
       - Update SenderReputation: `update_or_create(sender_address=thread.last_sender_address.lower())`, increment `spam_count` using F() expression, then `refresh_from_db()` to check auto-block threshold (ratio > 0.8 AND total >= 3 -> set is_blocked=True)
       - Create `ActivityLog(thread=thread, action=Action.SPAM_MARKED, performed_by=request.user, details=f"Marked as spam by {request.user}")`
       - Re-render detail panel with toast: `detail_context["toast_msg"] = "Marked as spam"`

       `mark_not_spam(request, pk)`:
       - Same pattern as mark_spam but `user_verdict=False`, `is_spam=False`
       - Decrement `spam_count` using `Greatest(F("spam_count") - 1, 0)` (never go below 0)
       - If sender `is_blocked=True`: auto-create `SpamWhitelist(entry=sender, entry_type="email", added_by=request.user, reason="Auto-whitelisted: user marked not-spam on blocked sender")` and set `is_blocked=False` (per SPAM-05 locked decision)
       - ActivityLog with `SPAM_UNMARKED`
       - Toast: "Marked as not spam" (+ "and sender unblocked" if was blocked)

       `undo_spam_feedback(request, pk)`:
       - `pk` is SpamFeedback ID
       - Delete the feedback, reverse the reputation change (decrement spam_count if was spam, increment if was not-spam)
       - Reverse Email.is_spam to original_verdict
       - Re-check auto-block status
       - Return re-rendered detail panel

    2. **URLs in `apps/emails/urls.py`** -- add 3 routes:
       - `threads/<int:pk>/mark-spam/` -> `mark_spam`
       - `threads/<int:pk>/mark-not-spam/` -> `mark_not_spam`
       - `spam-feedback/<int:pk>/undo/` -> `undo_spam_feedback`

    3. **Pipeline in `apps/emails/services/pipeline.py`**:
       - Add `_is_blocked(sender_email: str) -> bool` following `_is_whitelisted` pattern: query `SenderReputation.objects.filter(sender_address__iexact=sender_email, is_blocked=True).exists()`
       - In `process_single_email`, insert block check AFTER whitelist check but BEFORE spam filter (per locked decision: whitelist -> block check -> spam filter -> AI). If blocked, log at info level and return None (silent block, no ActivityLog per decision).
       - After `save_email_to_db` call, add SenderReputation tracking: `update_or_create(sender_address=email_msg.sender_email.lower())`, increment `total_count` via F(), and if `is_spam` also increment `spam_count`. Use `refresh_from_db()` before checking auto-block threshold.

    **CRITICAL pitfalls to avoid:**
    - Always `refresh_from_db()` after F() expression saves before reading `spam_ratio` (Pitfall 1 from research)
    - Use `update_or_create` not `get_or_create` for SenderReputation to avoid race conditions (Pitfall 2)
    - Wrap reputation updates in `transaction.atomic()` for undo safety (Pitfall 5)

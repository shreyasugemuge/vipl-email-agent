# Phase 4.5 Plan 02: Dead Code Removal and Activity Color Mappings

Removed unreachable DashboardView class and dashboard.html template, added color mappings for all Phase 3 and Phase 4 ActivityLog action types to the status_base template filter.

## Changes Made

### 1. Removed DashboardView (dead code)

- **apps/accounts/views.py**: Replaced with empty module containing only a docstring explaining DashboardView was removed (root RedirectView at /accounts/dashboard/ intercepts before accounts URLs are reached).
- **apps/accounts/urls.py**: Removed the `path("dashboard/", ...)` entry and the `from . import views` import. Only login and logout paths remain.
- **templates/accounts/dashboard.html**: Moved to `to_delete/dashboard.html` per CLAUDE.md rules (never truly delete files).

The redirect at `/accounts/dashboard/` still works via the root `config/urls.py` RedirectView, confirmed by the existing test `test_old_dashboard_redirects_to_emails`.

### 2. Added Activity Timeline Color Mappings

Updated `STATUS_BASE` in `apps/emails/templatetags/email_tags.py` to include all ActivityLog action types:

| Action | Color | Rationale |
|--------|-------|-----------|
| assigned | blue | Manual assignment, matches "new" |
| reassigned | amber | Attention-drawing change |
| status_changed | violet | Matches "acknowledged" |
| auto_assigned | cyan | System action, cool tone |
| claimed | teal | Self-service, distinct from assigned |
| sla_breached | red | Urgent/warning |
| priority_bumped | orange | Escalation |
